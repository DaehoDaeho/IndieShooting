오늘의 목표
- 적을 처치할 때 점수 증가
- 유니티 UI 시스템을 파악
- 게임 매니저(Score 관리 등등)
- UI와 코드를 연결하는 실습

변수 : 값을 저장하는 메모리 공간. 메모리의 별칭
 - 메모리는 주소를 가지고 있다. 0x0000ffff, 0xffff1111

변수의 구성 요소 : 접근권한지정자 자료형 변수이름 = 초기화 시킬 값;
접근권한지정자 (public, private, protected)
- public : 외부에서 접근이 가능함
- private : 외부에서 접근 불가. 해당 클래스 내에서만 접근이 가능

함수
- 클래스 내에서 특정 동작을 처리하는 기능을 묶어서 부르는 이름
함수의 형태
접근권한지정자 반환형 함수이름(파라미터)
{
}

Canvas
 - 화면 위에 UI를 표시하는 도화지와 같은 오브젝트

EventSystem
 - 버튼 클릭, 스크롤과 같은 사용자의 입력 감지를 처리해 주는 오브젝트

Anchor : 닻
- UI를 화면의 어느 위치를 기준으로 고정을 할 것인지를 정해줌

Text (TextMeshPro)
 - 텍스트를 출력하는 UI


GameManager
 - 게임 전체의 흐름과 상태를 관리
  예 : 현재 게임의 진행 상태, 점수 관리

싱글턴 (Singleton)
- 게임에서 단 하나만 존재해야 하는 관리자를 만들 때 쓰는 설계법
- 다른 스크립트에서 언제 어디서든 접근 할 수 있도록 해주는 기능

textScore.gameObject.SetActive(false); -> 오브젝트 비활성화
textScore.gameObject.SetActive(true); -> 오브젝트 활성화

Time.timeScale = 0; -> 모든 실행을 멈춘다
